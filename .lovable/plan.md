## Goals

1. Fix "Money generated by product × unit" not refreshing after sale edits.
2. Stop unit-casing duplicates (BAGS / Bags / bags) — enforce a canonical unit list everywhere.
3. Make the sales filter dropdowns for Products & Customers only show entries within the selected date range.
4. Give the Accountant a dense, at-a-glance summary page with a Weekly table (days as columns, products as rows) — one grid for bags, one for money — plus a Monthly report (bags sold, MoM diff per product, debts given, debts paid) all on one page.

Uses existing data — mostly UI work, minimal credits.

---

## 1. Money pivot not updating on edits

**Cause:** `DailyReport.tsx` reads `line_total` and falls back to `qty × unit_price`. When a sale is edited, `sales_items` rows are usually re-inserted, but if `line_total` was cached stale or the transaction subscription doesn't re-trigger the fetch, the money pivot lags behind the quantity pivot (which recomputes from qty).

**Fix:**
- Recompute money pivot from `quantity × unit_price` directly (ignore stored `line_total`) so it always matches the qty grid.
- Add a Supabase realtime subscription in `DailyReport` on `sales_items` + `sales_transactions` (filtered by date) so any edit re-fetches — same pattern as SalesTab.
- Verify the sales editor writes `unit_price` on every row update, not just on insert.

## 2. Canonical units — no more BAGS/Bags/bags duplicates

- Central list in `src/lib/units.ts` already defines `PIVOT_UNITS` (70kg, 50kg, 20kg, 10kg, 5kg, kg). Expose a `CANONICAL_UNITS` string array (`['bags','50kg','20kg','10kg','5kg','kg']`) as the single source of truth.
- Everywhere a user picks a unit (sales entry, sales edit dialog, bulk upload mapping, price manager, inventory), replace free-text/limited dropdown with a `Select` bound to `CANONICAL_UNITS`.
- On write, normalize with `canonicalUnitKey()` + `dbUnitForKey()` before insert/update so DB only ever stores canonical strings.
- In `DailyReport` pivot, group by `canonicalUnitKey(it.unit)` (case-insensitive) so legacy mixed-case rows collapse into one column.
- One-off migration: `UPDATE sales_items / inventory / product_prices SET unit = <canonical>` mapping case-insensitively — reversible, no data loss.

## 3. Date-scoped Products & Customers filters (SalesTab)

- In `SalesTab.tsx`, the product & customer dropdowns are currently built from all-time distinct values. Rebuild them from the already-filtered `transactions`/`items` slice (which respects the active date range).
- When the date range changes, dropdowns re-derive. If the currently selected product/customer is no longer in range, auto-clear that filter.

## 4. Accountant Summary Page

New route/tab inside `AccountantDashboard`: **Reports** with three stacked sections on one scrollable page. Each section shares one `PeriodPicker` at the top plus its own quick-preset chips.

### 4a. Weekly grid (days × products)

Two side-by-side tables (or toggle) inside one card:

```text
              Mon 1 | Tue 2 | Wed 3 | Thu 4 | Fri 5 | Sat 6 | Sun 7 | Total
High Yield    …bags | …     | …     | …     | …     | …     | …     | Σ
Layers        …     | …     | …     | …     | …     | …     | …     | Σ
…
Total         Σ     | Σ     | Σ     | Σ     | Σ     | Σ     | Σ     | ΣΣ
```

- Grid 1: Total bag-equivalents (uses `toBagEquivalent`).
- Grid 2: Money generated (Ksh).
- Row + column totals; grand total bottom-right.
- Colour intensity per cell (Tailwind `bg-primary/10 … /50`) proportional to that day's row max, so peaks pop at a glance.
- Sticky first column (product name) + sticky header.
- Default range: current week (Mon–Sun); PeriodPicker lets user pick any 7-day window (or Custom).

### 4b. Monthly report

Single card, three panels in a responsive grid:

- **Bags sold this month per product** — table: product | bags this month | bags last month | Δ absolute | Δ % (green/red).
- **Money this month per product** — same columns for revenue.
- **Debts snapshot** — 4 KPI tiles: Credit issued this month, Debt paid this month, Net debt change, Outstanding total. Below: top 10 debtors mini-table.

### 4c. KPI header (always visible)

Six compact tiles above the sections: Total bags (period), Total revenue, Cash collected, Credit issued, Debt paid, Active customers.
